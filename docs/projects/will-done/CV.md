## 핵심 기능 아키텍처

### **⏳ 지능형 타임 시프트 엔진 (Time-Shift Engine)**

단순 TODO 앱과 차별화되는 핵심 비즈니스 로직으로, 실시간 일정 변화를 계산해 자동 조정합니다.

**긴급 업무 대응**

긴급 태스크 발생 시 진행 중 태스크를 중단(PENDING) 및 분할하고, 이후 모든 일정을 자동 시프트

**언플러그드 타임 회피**

점심시간, 정기 회의 등 고정 시간대를 설정하면 스케줄러가 태스크 블록을 분할 배정

**완료/연장 반영**

조기 완료 시 일정 당김(Pull up), 지연 완료 시 일정 밀기(Push back)

### **🤖 AI 회고 및 Brag Document 자동화**

**성과 보고서 자동 생성**

완료 태스크의 계획 메모와 리뷰 메모를 취합해 마크다운 성과 문서 생성

**다중 모델 폴백(Fallback) 엔진**

Gemini API Rate Limit(429) 또는 서버 에러 발생 시
`로컬 캐시 → flash-lite → flash → pro` 순으로 자동 전환 재시도

**다중 주기 지원**

일간/주간/월간 단위 회고 생성 및 조회

## 트러블 슈팅

### 하네스(Harness) 문제 해결

AI 에이전트 기반 개발 과정에서 “이해는 했는데 실행이 불안정해지는” 하네스 이슈가 반복적으로 발생했습니다.

**관찰된 증상**

1. 배치 처리 경향: 여러 태스크를 한꺼번에 처리하다 일부만 진행
2. 검증 없는 진행: 빌드가 깨진 상태로 다음 작업 진행
3. 문서-코드 불일치: 변경 후 레퍼런스 문서 업데이트 누락
4. 불완전한 커밋: 커밋 단위가 섞이고 논리성이 깨짐

**대응**

- **문서 기반 SSOT(Single Source of Truth)**: `/docs/ai/STRUCTURE.md`를 “현재 구현 상태의 단일 기준”으로 두고 작업 전/후 동기화 규칙을 운영
- **단일 작업 루프**: `PLANNING.md`로 한 번에 하나의 작업만 진행하도록 강제
- **검증 게이트**: `cargo check` / `pnpm build` 통과 전에는 진행 금지
- **원자적 커밋**: 관련 파일만 스테이징하여 레이어/목적별 커밋 유지

## 프로젝트 회고

바이브 코딩에서 제 자신의 역할은 코드를 “대신 쓰는 것”이 아니라, **기획·설계·제약조건·검증 시스템**을 만드는 일이라고 정리했습니다.

1. 기획: 풀고 싶은 문제 정의 및 사용자 여정 설계
2. 설계: AI가 일관된 품질로 작업할 수 있는 레퍼런스 시스템 구축
3. 오케스트레이션: 도구별 강점에 맞춰 작업 분배
4. 품질 관리: 검증 루프와 커밋 규율로 실행을 통제
