## STAR

### 1. Situation (기존 시스템의 한계 및 배경)

- **단순 매칭 및 정규표현식의 성능 한계:** 대다수의 비속어 필터링 로직이 단순 문자열 포함 여부나 정규표현식을 사용함. 이는 금칙어 사전의 크기가 커질수록 모든 패턴을 개별적으로 검사해야 하므로 시간 복잡도가 선형적으로 증가(O(N × K))하는 성능 저하 문제를 내포함.
- **오탐(False Positive) 및 변칙 우회 문제:** 한국어 특성상 "시1발"과 같은 변칙 표현이나, "시발점"처럼 비속어가 포함되어 있지만 실제로는 정상 단어인 경우를 구분하지 못하는 라이브러리가 많아 직접 커스텀 필터링 엔진 구축의 필요성을 느낌.

### 2. Task (직면한 과제 및 목표)

- **기술 블로그 참고:** 배달의 민족의 기술 블로그에 나와있는 아호코라식 알고리즘을 사용한 비속어 필터에 대한 아이디어를 얻어 실제 라이브러리로 개발.
- **다중 패턴 매칭 최적화:** 수천 개의 비속어 패턴을 동시에 검색하면서도 입력 문자열의 길이에만 비례하는 시간 복잡도(O(N))를 가진 고성능 검색 엔진 구현.
- **범용 라이브러리 설계:** Java, Kotlin, Spring Boot 환경에서 모두 사용 가능하며, 사용자가 직접 금칙어를 추가/제외하거나 예외 단어(White-list)를 설정할 수 있는 유연한 API 제공.

### 3. Action (기술적 의사결정 및 행동)

- **Trie 자료구조 구축 및 한계 분석:** 문자열 집합을 트리 형태로 관리하는 Trie를 우선 구현했으나, 공통 접두사를 반복 탐색하는 비효율을 식별. 이를 통해 대규모 패턴 매칭에 최적화된 알고리즘을 탐색함.
- **Aho-Corasick 알고리즘 도입:** Trie에 '실패 함수(Failure Function)' 개념을 결합한 아호코라식 알고리즘을 핵심 엔진으로 채택. 매칭 실패 시 루트로 돌아가지 않고 최장 공통 접미사 노드로 바로 이동하게 설계하여, 입력 문자열을 단 한 번만 순회하면서 모든 금칙어를 동시 탐색하도록 최적화.
- **오탐 방지 및 유연한 예외 처리 로직 구현:** "시발점"과 같이 비속어를 포함한 정상 단어를 필터링에서 제외할 수 있도록 `ProfanityValidator` 내부에 화이트리스트(White-list) 기반 예외 처리 로직을 설계하여 정확도를 높임.
- **오픈소스 배포 및 문서화:** Gradle 및 Maven 환경에서 즉시 도입 가능하도록 Jitpack을 통해 라이브러리를 배포하고, Java/Kotlin 각각의 사용 예시를 포함한 README를 작성하여 커뮤니티 기여 환경 구축.

### 4. Result (정량적/정성적 성과)

- **검색 효율 극대화:** 아호코라식 도입을 통해 금칙어 개수와 상관없이 입력 문자열 길이에 비례하는 O(N)의 검색 속도를 달성, 기존 단순 Trie 방식 대비 비약적인 성능 향상.
- **실무 중심의 고도화:** 단순 알고리즘 구현을 넘어 '제외 단어 설정' 등 비즈니스 현장에서 필요한 기능들을 라이브러리에 녹여내어 실제 서비스 적용 가능한 수준의 도구로 고도화 중.
- **지식의 실체화:** 알고리즘적 이론(Trie, Aho-Corasick)을 실제 문제 해결(비속어 필터링)에 적용하여 오픈소스 라이브러리화함으로써, 기술적 깊이와 실행력을 동시에 증명.
